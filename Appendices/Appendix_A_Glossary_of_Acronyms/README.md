# Appendix A: Glossary of Acronyms

## Introduction

This appendix defines every acronym used in the handbook. Acronyms are organized by domain: cellular networking, Wi-Fi networking, legal, and general. Use this as a quick reference when reading technical chapters.

## Cellular Network Acronyms

| Acronym | Full Name | Definition |
|---|---|---|
| 3GPP | 3rd Generation Partnership Project | The standards organization that develops cellular protocols (4G LTE, 5G NR). All mobile network specifications are published as 3GPP technical specifications (TS) and reports (TR). |
| 4G LTE | 4th Generation Long Term Evolution | The fourth-generation cellular network standard. Deployed globally from approximately 2010 onward. Uses OFDMA on the downlink and SC-FDMA on the uplink. |
| 5G NR | 5th Generation New Radio | The fifth-generation cellular network standard. Introduces higher frequencies (mmWave), lower latency, and new positioning capabilities. Deployed from approximately 2020 onward. |
| 5G SA | 5G Standalone | 5G deployment using a 5G core network (5GC) without relying on 4G LTE infrastructure. Required for advanced 5G features including Multi-RTT positioning. |
| 5G NSA | 5G Non-Standalone | 5G deployment that relies on an existing 4G LTE core network (EPC) for control signaling while using 5G NR for data. Does not support 5G-specific positioning features. |
| AMF | Access and Mobility Management Function | The 5G core network function that handles registration, connection management, and mobility. Replaces the MME from 4G. Interfaces with the RAN via NGAP. |
| AoA | Angle of Arrival | A positioning method where the tower uses multiple directional antennas to determine the angle from which a phone's signal arrives. Combined with Timing Advance, provides 50–200 meter accuracy. |
| APN | Access Point Name | Identifies the packet data network a phone connects to (e.g., `internet`, `ims`, `enterprise.vpn`). Reveals the type of service being used. |
| CALEA | Communications Assistance for Law Enforcement Act | US law requiring telecommunications carriers to design their networks to support lawful interception. Mandates that carriers can provide real-time call content and metadata to law enforcement upon court order. |
| CDR | Call Detail Record | A log entry generated by a carrier for each call or SMS. Contains IMSI, IMEI, Cell ID, Timing Advance, timestamp, and the other party's number. Retained for months or years. |
| DL-AoD | Downlink Angle of Departure | A 5G positioning method where the phone measures the angle of reference signals transmitted by multiple tower beams. Provides 5–20 meter accuracy. |
| ECGI | E-UTRAN Cell Global Identifier | The unique identifier for a 4G LTE cell sector. Combines the PLMN ID (MCC + MNC) with the Cell Identity. Used in RRC and S1AP signaling. |
| E-CID | Enhanced Cell ID | A positioning method that combines Cell ID with Timing Advance and Angle of Arrival for improved accuracy (50–200 meters). |
| eNB | Evolved Node B | The 4G LTE base station (tower). Handles radio resource management, scheduling, and measurement collection. Communicates with the core network via S1AP. |
| gNB | Next Generation Node B | The 5G NR base station (tower). Supports higher frequencies, massive MIMO, and beamforming. Communicates with the 5G core via NGAP. |
| GUTI | Globally Unique Temporary Identifier | A temporary identifier assigned by the network to hide the IMSI after initial attach. Used in 4G. Replaced periodically to reduce tracking. |
| IMEI | International Mobile Equipment Identity | A unique 15-digit number that identifies the physical device hardware. Independent of the SIM card. Can be used to track a device even if the SIM is changed. Often called the "device fingerprint." |
| IMS | IP Multimedia Subsystem | The framework for delivering voice and multimedia services over IP networks. Used for VoLTE (Voice over LTE) and VoNR (Voice over NR). SIP signaling within IMS can leak call metadata. |
| IMSI | International Mobile Subscriber Identity | A unique 15-digit number stored on the SIM card that identifies the subscriber to the network. In 4G, transmitted in plain text during initial attach. In 5G, replaced by SUPI/SUCI. Links directly to billing identity. |
| LPP | LTE Positioning Protocol | A 3GPP protocol for positioning measurements between the phone and the network. Used for OTDOA, ECID, and other location methods. |
| LPPa | LTE Positioning Protocol Annex | A protocol for positioning measurements between the eNB and the location server (E-SMLC). Carries measurement data not available via LPP alone. |
| MCC | Mobile Country Code | The first three digits of the IMSI that identify the subscriber's home country (e.g., 310 for USA, 234 for UK, 460 for China). |
| MME | Mobility Management Entity | The 4G LTE core network function that handles authentication, mobility tracking, and session management. Communicates with eNBs via S1AP. |
| MNC | Mobile Network Code | The next two or three digits of the IMSI that identify the specific carrier within a country (e.g., 260 for T-Mobile USA, 410 for AT&T). |
| Multi-RTT | Multi-Cell Round Trip Time | A 5G positioning method where the phone exchanges timing requests with multiple towers simultaneously. Provides 1–10 meter accuracy. Requires 5G SA. |
| NAS | Non-Access Stratum | The signaling layer between the phone and the core network (MME in 4G, AMF in 5G). Handles authentication, attach/detach, and session management. Contains identity and mobility information. |
| NCGI | NR Cell Global Identifier | The unique identifier for a 5G NR cell sector. Similar to ECGI but for 5G. Used in RRC and NGAP signaling. |
| NGAP | Next Generation Application Protocol | The signaling protocol between the 5G gNB and the 5G core network (AMF). Carries NAS messages, initial context setup, and handover signaling. |
| OTDOA | Observed Time Difference of Arrival | A positioning method where the phone measures the arrival time difference of reference signals from multiple towers. Provides 10–50 meter accuracy. Passive — does not require phone cooperation. |
| PEI | Permanent Equipment Identifier | The 5G equivalent of IMEI. Identifies the physical device hardware. Used in 5G registration and lawful intercept. |
| PLMN | Public Land Mobile Network | A network operated by a single carrier in a specific country. Identified by the combination of MCC and MNC. |
| PSAP | Public Safety Answering Point | The call center that handles emergency calls (911 in the US, 112 in the EU). Receives E911 location data from the carrier. Retains call records and location data. |
| QCI | QoS Class Identifier | A value in LTE that tells the network what type of data is being transmitted (e.g., QCI=1 for voice, QCI=5 for IMS signaling, QCI=9 for web browsing). Reveals application type to the network. |
| QoS | Quality of Service | Parameters that define the priority, latency, and reliability requirements for a data flow. Used by the network to allocate resources and identify traffic types. |
| RRC | Radio Resource Control | The signaling layer between the phone and the base station (eNB/gNB). Handles connection establishment, measurement reports, and handovers. Contains Cell ID, signal strength, and neighbor cell measurements. |
| S1AP | S1 Application Protocol | The signaling protocol between the 4G eNB and the MME. Carries NAS messages, initial UE context setup, and handover signaling. A key forensic intercept point. |
| SCTP | Stream Control Transmission Protocol | The transport protocol used for S1AP (between eNB and MME) and NGAP (between gNB and AMF). Reliable, message-oriented, and multi-streamed. |
| SIP | Session Initiation Protocol | The signaling protocol for IMS voice calls. The SIP INVITE message contains dialed number, calling number, and call duration — even if the call audio is encrypted. |
| SUCI | Subscription Concealed Identifier | In 5G, the encrypted version of the SUPI (IMSI). The phone encrypts the IMSI using the home network's public key. Prevents passive IMSI capture by Stingray devices. |
| SUPI | Subscription Permanent Identifier | The 5G equivalent of IMSI. A globally unique identifier for the subscriber. In 5G, should only be transmitted in encrypted form (as SUCI) except during lawful intercept. |
| TA | Timing Advance | A measurement of signal round-trip time between the phone and the tower. Each TA unit equals approximately 78 meters in 4G LTE and 39 meters in 5G NR. Used for distance estimation in forensic analysis. |
| TDOA | Time Difference of Arrival | A positioning method that compares the arrival time of a signal at multiple towers to calculate the phone's location. Requires three or more synchronized towers. |
| TMSI | Temporary Mobile Subscriber Identity | A temporary identifier assigned by the network to replace the IMSI for privacy. In 4G, the equivalent is the GUTI. Periodically re-assigned to prevent long-term tracking. |
| UDM | Unified Data Management | The 5G core network function that stores subscriber data (SUPI, authentication credentials, service profile). Replaces the HSS from 4G. Interfaces with other core functions via service-based interfaces. |
| UE | User Equipment | The 3GPP term for the phone or mobile device. Encompasses both the mobile equipment (ME) and the SIM card (UICC). |
| UL-AoA | Uplink Angle of Arrival | A 5G positioning method where multiple towers measure the angle of the phone's uplink signal. Provides 5–20 meter accuracy using beamforming. |
| UTDOA | Uplink Time Difference of Arrival | A positioning method where towers measure the arrival time of the phone's uplink signal at multiple synchronized locations. Provides 5–30 meter accuracy. |

## Wi-Fi and Networking Acronyms

| Acronym | Full Name | Definition |
|---|---|---|
| AP | Access Point | A device that creates a wireless local area network (Wi-Fi network). Identified by its BSSID (MAC address) and SSID (network name). |
| BSSID | Basic Service Set Identifier | The MAC address of a Wi-Fi access point. Used in Wi-Fi geolocation databases to map routers to physical locations. Globally unique. |
| DHCP | Dynamic Host Configuration Protocol | A protocol that assigns IP addresses to devices on a network. The DHCP request can leak device hostname, vendor class ID, and MAC address. |
| DNS | Domain Name System | The protocol that translates domain names (signal.org) to IP addresses. DNS queries are visible to ISPs and network observers unless encrypted (DoH/DoT). |
| DoH | DNS over HTTPS | Encrypted DNS queries sent over HTTPS. Prevents ISPs from seeing which domains you visit. Supported by most modern browsers and OS configurations. |
| DoT | DNS over TLS | Encrypted DNS queries sent over a TLS connection. Similar to DoH but uses a dedicated port (853) rather than HTTPS port 443. |
| DPI | Deep Packet Inspection | Network analysis technique that examines packet contents beyond headers. ISPs use DPI to identify application types (VPN, Tor, Signal, streaming). |
| EAPOL | Extensible Authentication Protocol over LAN | The protocol used for Wi-Fi authentication (WPA2/WPA3 handshake). The 4-way handshake exchanges PMKID and other authentication materials. |
| ECH | Encrypted Client Hello | A TLS extension that encrypts the SNI (Server Name Indication) so that the destination domain is not visible during the TLS handshake. Rarely deployed as of 2026. |
| IPFIX | Internet Protocol Flow Information Export | A standard for exporting network flow information (source/destination IPs, ports, bytes, timestamps). Used by ISPs for traffic analysis. Used with NetFlow. |
| MAC | Media Access Control | A hardware address assigned to network interfaces. Wi-Fi MAC addresses are globally unique and can be used to identify specific devices. |
| NetFlow | Network Flow | A Cisco protocol for collecting IP traffic flow information. Records source/destination IPs, ports, protocol, and timestamps for every connection. |
| OUI | Organizationally Unique Identifier | The first three bytes (6 hex digits) of a MAC address that identify the manufacturer or vendor of the network interface. |
| PMKID | Pairwise Master Key Identifier | An identifier used in the WPA2/WPA3 4-way handshake. Can be captured by an attacker and used for offline brute-force attacks against weak Wi-Fi passwords. |
| SNI | Server Name Indication | A TLS extension that tells the server which domain name the client wants to connect to. Transmitted in plain text before encryption is established. Visible to ISPs and network monitors. |
| SSID | Service Set Identifier | The human-readable name of a Wi-Fi network (e.g., "HomeNetwork", "CoffeeShopWiFi"). Leaked by probe requests when a phone searches for remembered networks. |
| VLAN | Virtual Local Area Network | A logical subdivision of a network. Some ISPs use VLANs to separate customer traffic. Relevant for forensic traceability. |
| VPN | Virtual Private Network | Encrypted tunnel between your device and a server. Shifts trust from your ISP to the VPN provider. Hides your IP and encrypts traffic content, but does not prevent metadata analysis. |

## Legal Acronyms

| Acronym | Full Name | Definition |
|---|---|---|
| CALEA | Communications Assistance for Law Enforcement Act | US law requiring carriers to support lawful interception. See main entry in Cellular section above. |
| CPNI | Customer Proprietary Network Information | Data about a customer's telecommunications service usage protected under US law. Includes call records, service features, and billing information. |
| ECPA | Electronic Communications Privacy Act | US law governing government access to electronic communications. Includes the Stored Communications Act (SCA) and the Wiretap Act (Title III). |
| FISA | Foreign Intelligence Surveillance Act | US law governing surveillance for foreign intelligence purposes. Allows warrants from the FISA court with lower standards than Title III. |
| FOIA | Freedom of Information Act | US law allowing citizens to request records from federal agencies. Used by researchers to obtain information about surveillance programs and data retention policies. |
| LE | Law Enforcement | Generic term for police, federal agents, or any government entity with investigative authority. |
| NSL | National Security Letter | An administrative subpoena used by US intelligence agencies to obtain subscriber records without judicial oversight. Includes a gag order preventing the recipient from disclosing the request. |
| PR/TT | Pen Register / Trap and Trace | A legal order that allows law enforcement to obtain real-time metadata (dialed numbers, duration) for a specific phone. Lower legal standard than a wiretap. |
| SCA | Stored Communications Act | Title II of ECPA. Governs government access to stored communications (email, messages, records) held by third-party providers (ISPs, email services, messaging platforms). §2703(d) orders are issued under this act. |
| Title III | Title III of the Omnibus Crime Control and Safe Streets Act | US law governing real-time interception of communications content (wiretaps). Requires probable cause and a court order. Standards are stricter than for metadata or stored records. |

## General Acronyms

| Acronym | Full Name | Definition |
|---|---|---|
| E2EE | End-to-End Encryption | Encryption where only the communicating parties can decrypt messages. Intermediaries (servers, carriers) cannot read the content. Signal, WhatsApp, and iMessage use E2EE. |
| EXIF | Exchangeable Image File Format | Metadata embedded in digital photos, including camera model, GPS coordinates, timestamp, and device serial number. A forensic goldmine for identifying devices and locations. |
| FCM | Firebase Cloud Messaging | Google's push notification service for Android. Runs as part of Google Play Services. Can transmit device location and identity data. Disabled on de-Googled devices. |
| GLS | Google Location Services | Google's location database combining Wi-Fi BSSID, cell tower, and GPS data. Every Android device with Google Play Services contributes to and queries this database. |
| HUMINT | Human Intelligence | Intelligence gathered from human sources (informants, social engineering). Defeats all technical privacy protections. |
| ML | Machine Learning | Statistical models used to classify and predict patterns. Applied to traffic analysis for identifying application usage from encrypted traffic. |
| OpSec | Operations Security | The process of protecting sensitive information by identifying and controlling unclassified indicators. Often abbreviated as operational security. |
| PoC | Proof of Concept | A demonstration that a concept or theory is feasible. In security research, a PoC exploit demonstrates that a vulnerability is real. |
| RF | Radio Frequency | Electromagnetic radiation used for wireless communication. Includes cellular, Wi-Fi, Bluetooth, GPS, and NFC frequencies. |
| SDR | Software-Defined Radio | A radio system where components typically implemented in hardware (mixers, filters, modulators) are implemented in software. Used for capturing and analyzing wireless signals. |
| SIM | Subscriber Identity Module | The smart card that stores IMSI and authentication keys for cellular network access. Contains the subscriber identity that links to billing and personal information. |
| SoC | System on Chip | An integrated circuit containing most components of a computer or electronic system on a single chip. Modern phone SoCs include CPU, GPU, baseband, Wi-Fi, and Bluetooth on one die. |
| TTL | Time to Live | A field in IP packets that limits the number of hops before the packet is discarded. Can be used for OS fingerprinting. |
| UICC | Universal Integrated Circuit Card | The technical term for the SIM card. A smart card that runs a Java Card operating system and stores subscriber credentials. |
